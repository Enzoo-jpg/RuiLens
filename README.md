# 艾伯维 · 乌帕替尼(瑞福) 患者池分析看板

基于销售明细底表（`销售明细查询报表_oneid.xlsx`）的 Streamlit 在线分析工具，第一版聚焦「患者池累计 + 患者分布」。

## 已确认口径
- **患者唯一键**：`oneid + 开票抬头`（两者皆空时回退 `会员号` / `会员姓名`）
- **首次购药时间**：每个患者最小的 `销售时间`
- **当月新患**：首次购药时间落在当前统计月的患者数
- **YTD 老患**：首次购药时间落在 `[YTD 开始日, 上月末]` 的累计患者数（= 截至上月底累计池，不要求当月复购）
- **城市**：`药房名称` 前 2 字
- **适应症**：每个患者**末次（最近）购药**的适应症，经清洗映射表归到标准适应症
- **当月患者（分布图）**：当月有销售记录的患者，按末次购药时的药房 / 适应症归类

## 本地运行
```bash
pip install -r requirements.txt
streamlit run app.py
```
应用默认读取本地路径 `D:/工作/艾伯维/26-7月/销售明细查询报表_oneid.xlsx`；
也可在侧边栏「上传销售明细表」直接上传，无需硬编码路径。

## 部署（Streamlit Cloud）
1. 将本仓库推送到 GitHub（**切勿提交 xlsx 数据文件，已写入 .gitignore**）。
2. 在 [Streamlit Cloud](https://streamlit.io/cloud) 关联该仓库，入口 `app.py`。
3. 云端无本地数据，请使用页面上的「上传销售明细表」上传数据。

## 隐私提示
底表含患者电话、姓名、`oneid` 等敏感个人信息，**请勿将 xlsx 提交进任何仓库**；
本仓库仅包含代码，数据通过上传交互传入、不上云存储。
